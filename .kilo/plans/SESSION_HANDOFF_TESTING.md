## 5. Рекомендации по инструменту

**Playwright** — лучший выбор для этого проекта:

- Уже в экосистеме (упоминается в AGENTS.md)
- Поддерживает multi-page, multi-context (можно симулировать нескольких пользователей одновременно)
- Встроенная запись видео/трейсов
- API-тесты и UI-тесты в одном фреймворке
- `ignoreHTTPSErrors: true` решит проблему с сертификатом Let's Encrypt

**Альтернатива**: k6 (Grafana) — для нагрузочного тестирования API (RPS, latency). Не для UI.

## 7. Что НЕ сделано / требует внимания

1. **Тестов нет** — `npm test` не настроен (vitest работает только через `npx vitest run`)
2. **Playwright не установлен** — нужно `npm install -D @playwright/test`
3. **БД на сервере пустая** — выполните шаги из раздела 6 для наполнения тестовыми данными
4. **Step12Publish** в визарде при редактировании показывает дублирующиеся кнопки (внутренняя + навбар) — косметический баг
5. **TLS-сертификат** — на Windows машине пользователя ошибка `CRYPT_E_NO_REVOCATION_CHECK` при curl/pwsh, браузер работает
6. **Статус published** — нужна оплата через Т-Банк для перехода в pending_review → published. Для тестов можно вручную выставить статус через PATCH/docker exec
